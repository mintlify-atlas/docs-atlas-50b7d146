## Overview

TradeMaster Transactions uses **React Router 6.3** for declarative routing with nested layouts, protected routes, and code splitting. The routing system supports three layout types: authenticated full layout, guest blank layout, and error pages.

## Route Configuration

All routes are defined in `src/routes/Router.js` using React Router's object-based configuration.

### Basic Structure

```jsx src/routes/Router.js
import { Navigate } from 'react-router-dom';
import AuthGuard from 'src/guards/authGuard/AuthGuard';
import GuestGuard from 'src/guards/authGuard/GuestGaurd';
import PermissionGuard from 'src/guards/authGuard/PermissionGuard';

const Router = [
  {
    path: '/',
    element: (
      <AuthGuard>
        <FullLayout />
      </AuthGuard>
    ),
    children: [
      // Protected routes
    ],
  },
  {
    path: '/auth',
    element: (
      <GuestGuard>
        <BlankLayout />
      </GuestGuard>
    ),
    children: [
      // Authentication routes
    ],
  },
];

export default Router;
```

### Using Routes in App

```jsx src/App.jsx
import { useRoutes } from 'react-router-dom';
import Router from './routes/Router';

function App() {
  const routing = useRoutes(Router);
  
  return (
    <ThemeProvider theme={theme}>
      {routing}
    </ThemeProvider>
  );
}
```

## Lazy Loading

All route components are lazy-loaded using React.lazy() wrapped in a custom `Loadable` component.

### Loadable Wrapper

```jsx src/layouts/full/shared/loadable/Loadable.jsx
import React, { Suspense } from 'react';
import Spinner from 'src/views/spinner/Spinner';

const Loadable = (Component) => (props) => (
  <Suspense fallback={<Spinner />}>
    <Component {...props} />
  </Suspense>
);

export default Loadable;
```

### Creating Lazy Routes

```jsx Route Lazy Loading
import { lazy } from 'react';
import Loadable from '../layouts/full/shared/loadable/Loadable';

// Lazy load components
const ModernDash = Loadable(
  lazy(() => import('../views/dashboard/Modern'))
);
const StaffTable = Loadable(
  lazy(() => import('../views/Users/Staff/StaffTable'))
);
const ContractsTable = Loadable(
  lazy(() => import('../views/Contracts/ContractsTable'))
);
```

<Note>
Lazy loading splits code into separate chunks, reducing initial bundle size and improving load time.
</Note>

## Navigation

### Programmatic Navigation

Use the `useNavigate` hook for programmatic navigation:

```jsx useNavigate Example
import { useNavigate } from 'react-router-dom';

function MyComponent() {
  const navigate = useNavigate();

  const handleViewDetails = (staffId) => {
    navigate('/usuarios-detalle-staff', { 
      state: { staffId } 
    });
  };

  const handleBack = () => {
    navigate(-1); // Go back
  };

  return (
    <div>
      <button onClick={() => handleViewDetails('123')}>
        View Details
      </button>
      <button onClick={handleBack}>Back</button>
    </div>
  );
}
```

### Link Component

Use the `Link` component for declarative navigation:

```jsx Link Example
import { Link } from 'react-router-dom';

function Navigation() {
  return (
    <nav>
      <Link to="/dashboards/modern">Dashboard</Link>
      <Link to="/usuarios-staff">Staff</Link>
      <Link to="/eventos">Events</Link>
    </nav>
  );
}
```

### Passing State

Pass state between routes using the `state` option:

```jsx Passing State
// Navigate with state
navigate('/usuarios-detalle-staff', { 
  state: { 
    staffId: '123',
    from: 'staff-list' 
  } 
});

// Receive state
import { useLocation } from 'react-router-dom';

function DetailPage() {
  const location = useLocation();
  const { staffId, from } = location.state || {};

  return <div>Staff ID: {staffId}</div>;
}
```

## Route Protection

### Authentication Guard

All authenticated routes are wrapped in `AuthGuard`:

```jsx src/guards/authGuard/AuthGuard.js
const AuthGuard = ({ children }) => {
  const { isAuthenticated } = useAuth();
  const navigate = useNavigate();

  useEffect(() => {
    if (!isAuthenticated) {
      navigate('/auth/login', { replace: true });
    }
  }, [isAuthenticated, navigate]);

  return children;
};
```

### Guest Guard

Authentication routes use `GuestGuard` to redirect authenticated users:

```jsx src/guards/authGuard/GuestGuard.js
const GuestGuard = ({ children }) => {
  const { isAuthenticated } = useAuth();
  const navigate = useNavigate();

  useEffect(() => {
    if (isAuthenticated) {
      navigate('/', { replace: true });
    }
  }, [isAuthenticated, navigate]);

  return children;
};
```

### Permission Guard

Individual routes use `PermissionGuard` for role-based access:

```jsx Using PermissionGuard
<PermissionGuard action="view" subject="ViewStaff">
  <StaffTable />
</PermissionGuard>
```

See [Guards Documentation](/architecture/guards) for detailed information.

## Best Practices

<Steps>
  <Step title="Use Permission Guards">
    Always wrap protected routes in PermissionGuard
  </Step>
  <Step title="Lazy Load Components">
    Use React.lazy() for all route components
  </Step>
  <Step title="Pass State Carefully">
    Don't store sensitive data in navigation state
  </Step>
  <Step title="Handle 404s">
    Implement catch-all routes for undefined paths
  </Step>
  <Step title="Use Replace for Redirects">
    Use `replace: true` option for redirects to avoid back button issues
  </Step>
</Steps>
