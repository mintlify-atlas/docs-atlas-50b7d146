## Overview

TradeMaster Transactions uses **Redux Toolkit** for centralized state management with **redux-persist** for local storage persistence. The store is organized into feature-based slices, each managing its own domain of application state.

## Store Configuration

The Redux store is configured in `src/store/Store.js` with persistence and serialization middleware.

### Store Setup

```js src/store/Store.js
import { configureStore } from '@reduxjs/toolkit';
import { persistReducer, persistStore } from 'redux-persist';
import storage from 'redux-persist/lib/storage';

const persistConfig = {
  key: 'root',
  storage,
  whitelist: ['auth', 'customizer', 'setup']
};

const rootReducer = combineReducers({
  customizer: CustomizerReducer,
  setup: SetupReducer,
  auth: authReducer,
  staff: StaffReducer,
  // ... other reducers
});

const persistedReducer = persistReducer(persistConfig, rootReducer);

export const store = configureStore({
  reducer: persistedReducer,
  middleware: (getDefaultMiddleware) =>
    getDefaultMiddleware({
      serializableCheck: {
        ignoredActions: [FLUSH, REHYDRATE, PAUSE, PERSIST, PURGE, REGISTER],
      },
    }),
});

export const persistor = persistStore(store);
```

<Note>
Only `auth`, `customizer`, and `setup` slices are persisted to localStorage. Other slices are re-fetched on app load.
</Note>

### Middleware Configuration

The serializable check is configured to ignore redux-persist actions that contain non-serializable values:

- `FLUSH`: Clear persist cache
- `REHYDRATE`: Load persisted state
- `PAUSE`: Pause persistence
- `PERSIST`: Start persistence
- `PURGE`: Remove persisted state
- `REGISTER`: Register reducers

### Authentication State

The auth slice manages user authentication state and user profile:

```js src/store/apps/auth/authSlice.js
import { createSlice } from '@reduxjs/toolkit';
import moment from 'moment';

const initialState = {
  isAuthenticated: false,
  isInitialized: false,
  user: null,
};

const authSlice = createSlice({
  name: 'auth',
  initialState,
  reducers: {
    authStateChanged(state, action) {
      const { isAuthenticated, user } = action.payload;

      if(user !== null){
        const convertTimestamp = (timestamp) => {
          if (timestamp && timestamp.seconds) {
            const date = new Date(
              timestamp.seconds * 1000 + 
              timestamp.nanoseconds / 1000000
            );
            return moment(date).format('DD/MM/YYYY - hh:mm:ss a');
          }
          return null;
        };

        user.date.last_update = convertTimestamp(user.date?.last_update)
        user.date.last_access = convertTimestamp(user.date?.last_access)
      }

      state.isAuthenticated = isAuthenticated;
      state.isInitialized = true;
      state.user = user;
    },
  },
});

export const { authStateChanged } = authSlice.actions;
```

<Accordion title="User Object Structure">
The user object stored in auth state contains:
- `uid`: Firebase user ID
- `email`: User email address
- `name`: Display name
- `avatar`: Profile image URL
- `account_type`: Role (Administrador, Coordinador, etc.)
- `phone`: Phone number
- `address`: Address object with line, zipcode, city, country
- `date.last_access`: Last login timestamp (formatted)
- `date.last_update`: Last profile update timestamp
- `last_IP`: Last login IP address
</Accordion>

### Example Slice Implementation

```js Example Slice Pattern
import { createSlice } from '@reduxjs/toolkit';

const initialState = {
  items: [],
  loading: false,
  error: null,
  selectedItem: null,
};

const exampleSlice = createSlice({
  name: 'example',
  initialState,
  reducers: {
    setItems: (state, action) => {
      state.items = action.payload;
      state.loading = false;
    },
    setLoading: (state, action) => {
      state.loading = action.payload;
    },
    setError: (state, action) => {
      state.error = action.payload;
      state.loading = false;
    },
    selectItem: (state, action) => {
      state.selectedItem = action.payload;
    },
  },
});

export const { setItems, setLoading, setError, selectItem } = 
  exampleSlice.actions;
export default exampleSlice.reducer;
```

## Using Redux State

### Reading State

Use the `useSelector` hook to read state in components:

```jsx Component Example
import { useSelector } from 'react-redux';

function MyComponent() {
  const user = useSelector((state) => state.auth.user);
  const theme = useSelector((state) => state.customizer.activeTheme);
  const transactions = useSelector((state) => state.Transactions.items);

  return (
    <div>
      <h1>Welcome, {user?.name}</h1>
      <p>Theme: {theme}</p>
    </div>
  );
}
```

### Dispatching Actions

Use the `useDispatch` hook to dispatch actions:

```jsx Dispatching Actions
import { useDispatch } from 'react-redux';
import { setTheme, toggleSidebar } from 'src/store/customizer/CustomizerSlice';
import { authStateChanged } from 'src/store/apps/auth/authSlice';

function SettingsComponent() {
  const dispatch = useDispatch();

  const handleThemeChange = (theme) => {
    dispatch(setTheme(theme));
  };

  const handleToggleSidebar = () => {
    dispatch(toggleSidebar());
  };

  return (
    <div>
      <button onClick={() => handleThemeChange('DARK_THEME')}>
        Dark Mode
      </button>
      <button onClick={handleToggleSidebar}>
        Toggle Sidebar
      </button>
    </div>
  );
}
```

## Async Operations

For async operations like Firebase queries, use async thunks:

```js Async Thunk Example
import { createAsyncThunk, createSlice } from '@reduxjs/toolkit';
import { Firestore } from 'src/guards/firebase/Firebase';

// Define async thunk
export const fetchEvents = createAsyncThunk(
  'events/fetchEvents',
  async (clientId) => {
    const snapshot = await Firestore
      .collection('events')
      .where('client_id', '==', clientId)
      .get();
    
    return snapshot.docs.map(doc => ({
      id: doc.id,
      ...doc.data()
    }));
  }
);

// Handle in slice
const eventsSlice = createSlice({
  name: 'events',
  initialState: {
    events: [],
    loading: false,
    error: null,
  },
  reducers: {},
  extraReducers: (builder) => {
    builder
      .addCase(fetchEvents.pending, (state) => {
        state.loading = true;
        state.error = null;
      })
      .addCase(fetchEvents.fulfilled, (state, action) => {
        state.events = action.payload;
        state.loading = false;
      })
      .addCase(fetchEvents.rejected, (state, action) => {
        state.error = action.error.message;
        state.loading = false;
      });
  },
});
```

## State Persistence

### Persisted Slices

Three slices are persisted to localStorage:

1. **auth**: User authentication state
2. **customizer**: UI preferences
3. **setup**: Platform configuration

### Rehydration

The `PersistGate` component delays rendering until state is rehydrated:

```jsx src/main.jsx
<PersistGate loading={null} persistor={persistor}>
  <BrowserRouter>
    {/* App content */}
  </BrowserRouter>
</PersistGate>
```

### Clearing Persisted State

```js Clear Persistence
import { persistor } from 'src/store/Store';

// Purge all persisted state
await persistor.purge();

// Flush pending persistence
await persistor.flush();
```

## Best Practices

<AccordionGroup>
  <Accordion title="Keep slices focused">
    Each slice should manage a single domain of state. Don't create mega-slices.
  </Accordion>
  
  <Accordion title="Use immer's draft state">
    Redux Toolkit uses Immer, so you can write "mutating" logic that's actually immutable.
  </Accordion>
  
  <Accordion title="Normalize complex data">
    For relational data, consider normalizing with objects keyed by ID.
  </Accordion>
  
  <Accordion title="Selective persistence">
    Only persist critical state. Large data sets should be re-fetched.
  </Accordion>
  
  <Accordion title="Use selectors">
    Create memoized selectors with reselect for derived state.
  </Accordion>
</AccordionGroup>
