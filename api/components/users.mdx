## Overview

User components handle the management of different user types in the TradeMaster Transactions platform. Components are organized by user role in `src/components/apps/users/`.

## User Types

The platform supports four user types:

- **Clients** - Event organizers and promoters
- **Collaborators** - Staff assigned to events and offices
- **Customers** - Ticket purchasers and attendees
- **Staff** - Platform administrators and operators

## ClientsTableList

Displays a paginated table of client users with search and filtering.

**Location:** `src/components/apps/users/Clients/ClientsTableList.js:28`

### Features

- **Global Search** - Search by name, email, or account type
- **Status Filtering** - Filter by active/inactive status
- **Sortable Columns** - Sort by name, last access, status, or email
- **Pagination** - 10, 15, or 20 rows per page
- **CSV Export** - Download client data
- **Permission Controls** - Create button requires `create:usersClients` permission

### Data Fields

The table displays:

| Column | Field | Description |
|--------|-------|-------------|
| Nombre | `name` | Client name with avatar and account type |
| Último Acceso | `date.last_access` | Last login date |
| Estatus | `status` | Active/Inactive badge |
| Correo Electrónico | `email` | Client email address |
| Acciones | - | View details button |

### Usage Example

```jsx
import ClientsTableList from 'src/components/apps/users/Clients/ClientsTableList';

function ClientsPage() {
  return (
    <div>
      <Typography variant="h4">Client Management</Typography>
      <ClientsTableList />
    </div>
  );
}
```

### Redux Integration

The component connects to the Redux store:

```jsx
import { fetchClientsUsers } from 'src/store/apps/Users/ClientsSlice';

// Dispatched on component mount
dispatch(fetchClientsUsers());

// Select data from store
const clients = useSelector((state) => state.clients.Clients);
```

## Common Patterns

### User Data Structure

All user types share common fields:

```typescript
interface BaseUser {
  uid: string;              // Firebase Auth UID
  name: string;             // Display name
  email: string;            // Email address
  image?: string;           // Profile image URL
  status: boolean;          // Active/Inactive
  date: {
    created: Timestamp;     // Creation date
    last_access: string;    // Last login
    updated: Timestamp;     // Last update
  };
  account_type: string;     // User classification
}
```

### User Actions

Common actions available for user components:

- **View Details** - Navigate to user detail page
- **Edit** - Modify user information
- **Deactivate** - Disable user access
- **Activate** - Re-enable user access
- **Delete** - Remove user (admin only)

### Navigation Patterns

User components follow consistent navigation:

```jsx
const navigate = useNavigate();

// Navigate to client detail
navigate(`/usuarios-detalle-cliente?id=${clientId}`);

// Navigate to create client
navigate('/usuarios-clientes-crear');

// Navigate to edit client
navigate(`/usuarios-clientes-editar?id=${clientId}`);
```

## Styling and Theming

All user components use consistent Material-UI theming:

```jsx
const theme = useTheme();

// Status chip colors
const statusColor = status 
  ? theme.palette.success.main 
  : theme.palette.error.main;

// Background for dark mode
const bgColor = theme.palette.mode === 'dark' 
  ? 'rgba(255, 255, 255, 0.05)' 
  : 'rgba(0, 0, 0, 0.02)';
```
