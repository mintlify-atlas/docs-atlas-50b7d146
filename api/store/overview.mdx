## Introduction

The TradeMaster Transactions platform uses Redux Toolkit for centralized state management with Redux Persist for state persistence across sessions. The store combines multiple feature-specific slices into a unified state tree.

## Store Configuration

The main store is configured in `src/store/Store.js` using Redux Toolkit's `configureStore`.

### Basic Setup

```javascript
import { configureStore } from '@reduxjs/toolkit';
import storage from 'redux-persist/lib/storage';
import { combineReducers } from 'redux';
import { persistReducer, persistStore } from 'redux-persist';

export const store = configureStore({
  reducer: persistedReducer,
  middleware: (getDefaultMiddleware) =>
    getDefaultMiddleware({
      serializableCheck: {
        ignoredActions: [FLUSH, REHYDRATE, PAUSE, PERSIST, PURGE, REGISTER],
      },
    }),
});

export const persistor = persistStore(store);
```

## Persistence Configuration

The store uses Redux Persist to save specific slices to localStorage.

<ParamField path="key" type="string" default="root">
  The key used to store the persisted state in localStorage
</ParamField>

<ParamField path="storage" type="object">
  Storage engine (defaults to localStorage)
</ParamField>

<ParamField path="whitelist" type="array">
  Array of reducer keys to persist. Only `auth`, `customizer`, and `setup` are persisted.
</ParamField>

```javascript
const persistConfig = {
  key: 'root',
  storage,
  whitelist: ['auth', 'customizer', 'setup']
};
```

## Middleware Configuration

The store is configured with custom middleware settings to handle Redux Persist actions:

```javascript
middleware: (getDefaultMiddleware) =>
  getDefaultMiddleware({
    serializableCheck: {
      ignoredActions: [FLUSH, REHYDRATE, PAUSE, PERSIST, PURGE, REGISTER],
    },
  })
```

This configuration ignores serialization checks for Redux Persist actions, which contain non-serializable values.

## Usage

### Accessing the Store

```javascript
import { store } from './store/Store';

// Get current state
const state = store.getState();

// Access specific slice
const authState = state.auth;
const transactions = state.Transactions;
```

### With React Components

```javascript
import { useSelector, useDispatch } from 'react-redux';

function MyComponent() {
  const dispatch = useDispatch();
  const user = useSelector((state) => state.auth.user);
  const events = useSelector((state) => state.events.Events);
  
  // Component logic
}
```

## Best Practices

- **Immutable Updates**: All reducers use Redux Toolkit's Immer integration for immutable state updates
- **Async Operations**: Use thunks for async operations (Firebase queries, API calls)
- **Selectors**: Create memoized selectors for derived state
- **Error Handling**: Each slice includes error state for operation failures
- **Type Safety**: Consider adding TypeScript for enhanced type safety
