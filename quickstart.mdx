# Quick Start Guide

This guide will help you get up and running with TradeMaster Transactions (TMT) quickly. You'll set up your account, configure Firebase, and perform your first key operations.

<Note>
  Before starting, ensure you have completed the [installation](/installation) steps and have Node.js 16+ and npm installed.
</Note>

## Prerequisites

Before you begin, you'll need:

- Node.js 16+ and npm
- A Firebase project with Authentication and Firestore enabled
- Firebase service account credentials
- Basic knowledge of React and JavaScript

<Steps>
  <Step title="Configure Firebase">
    First, set up your Firebase configuration. Open `src/guards/firebase/Firebase.js` and update the Firebase config:

    ```javascript src/guards/firebase/Firebase.js
    const firebaseConfig = {
      apiKey: "your-api-key",
      authDomain: "your-project.firebaseapp.com",
      projectId: "your-project-id",
      storageBucket: "your-project.appspot.com",
      messagingSenderId: "your-messaging-sender-id",
      appId: "your-app-id",
      measurementId: "your-measurement-id"
    };
    ```

    <Warning>
      Never commit your Firebase credentials to version control. Use environment variables in production.
    </Warning>
  </Step>

  <Step title="Start the Development Server">
    Launch the development server:

    ```bash
    npm run dev
    ```

    The application will be available at `http://localhost:5173` (or the port shown in your terminal).

    <Note>
      The dev server uses Vite's hot module replacement (HMR) for instant updates during development.
    </Note>
  </Step>

  <Step title="Create Your First User">
    TMT uses Firebase Authentication with custom user validation. To create a staff user:

    1. **Add user to Firestore**: Create a document in the `u_staff` collection:

    ```javascript
    // Firestore document structure
    {
      email: "admin@example.com",
      name: "Admin User",
      status: true,
      account_type: "admin",
      phone: "+1234567890",
      address: {
        line: "123 Main St",
        zipcode: "12345",
        city: "New York",
        country: { name: "United States", code: "US" }
      },
      image: null,
      date: {
        created: serverTimestamp(),
        last_access: null
      }
    }
    ```

    2. **Create the Firebase Auth user** using the Firebase Console or Admin SDK
    
    3. **Sign in** at `http://localhost:5173/auth/login`

    The authentication flow validates users through the `validate_user_platform` Cloud Function:

    ```javascript src/views/authentication/authForms/AuthLogin.js
    validateUserPlatform({ platform: 'u_staff', email: values.email })
      .then(async (resp) => {
        if (resp.data.valido === true) {
          await signin(values.email, values.password);
        }
      });
    ```
  </Step>

  <Step title="Navigate the Platform">
    Once logged in, you'll see the main dashboard. The platform is organized into key sections:

    - **Dashboard**: Overview of platform metrics and recent activity
    - **Events**: Create and manage events
    - **Tickets**: View and manage tickets
    - **Offices**: Manage sales offices and portals
    - **Transactions**: Track financial transactions
    - **Users**: Manage staff, clients, and collaborators
    - **Settings**: Platform configuration

    <img
      src="/images/dashboard.png"
      alt="TMT Dashboard"
    />
  </Step>

  <Step title="Create Your First Event">
    Let's create an event to understand the core workflow:

    1. Navigate to **Events** → **New Event**
    2. Fill in the event details:

    ```javascript
    // Event data structure
    {
      name: "Summer Concert 2024",
      description: "Annual summer music festival",
      date: {
        start: Timestamp,
        end: Timestamp
      },
      venue: venueRef, // Reference to event venue
      status: "active",
      tickets: {
        total: 5000,
        available: 5000,
        sold: 0
      },
      pricing: [
        {
          category: "General Admission",
          price: 50.00,
          quantity: 3000
        },
        {
          category: "VIP",
          price: 150.00,
          quantity: 500
        }
      ]
    }
    ```

    3. Assign the event to a venue
    4. Configure ticket pricing and categories
    5. Click **Create Event**

    The event creation triggers the `create_event_seatsio` Cloud Function if seating charts are enabled.
  </Step>

  <Step title="Generate Tickets">
    After creating an event, generate tickets:

    1. Go to the event detail page
    2. Click **Generate Tickets**
    3. Specify the quantity and category
    4. Tickets are created with unique QR codes

    ```javascript
    // Ticket generation using Cloud Function
    import { tickets_generate } from 'src/guards/firebase/Firebase';

    const generateTickets = async (eventId, quantity, category) => {
      const result = await tickets_generate({
        eventId,
        quantity,
        category,
        price: 50.00
      });
      
      return result.data.tickets;
    };
    ```

    Each ticket includes:
    - Unique ticket ID
    - QR code for validation
    - Event reference
    - Status (available, sold, validated, cancelled)
    - Transaction history
  </Step>

  <Step title="Set Up an Office Portal">
    Create a sales office to enable distributed ticket sales:

    1. Navigate to **Offices** → **Portals**
    2. Click **New Portal**
    3. Configure the portal:

    ```javascript
    // Portal configuration
    {
      name: "Downtown Sales Office",
      location: {
        address: "456 Market St",
        city: "San Francisco",
        coordinates: {
          lat: 37.7749,
          lng: -122.4194
        }
      },
      status: "active",
      pin: "1234", // Access PIN for office
      assigned_events: [eventRef],
      collaborators: [userRef]
    }
    ```

    4. Assign collaborators to the office
    5. Allocate offline tickets for the office

    The portal system uses PIN authentication:

    ```javascript
    import { office_pin_set, helpers_event_pin } from 'src/guards/firebase/Firebase';

    // Set office PIN
    await office_pin_set({ officeId, pin: '1234' });

    // Validate office access
    await helpers_event_pin({ officeId, pin, eventId });
    ```
  </Step>

  <Step title="Process Your First Transaction">
    Record a ticket sale transaction:

    1. Navigate to **Transactions** → **New Transaction**
    2. Select the ticket(s) being sold
    3. Choose payment method
    4. Record customer information
    5. Complete the transaction

    The transaction is stored in Firestore and can be reconciled later:

    ```javascript
    // Transaction structure
    {
      id: "TXN-001",
      type: "sale",
      amount: 50.00,
      currency: "USD",
      payment_method: "card",
      tickets: [ticketRef],
      office: officeRef,
      collaborator: userRef,
      customer: {
        name: "John Doe",
        email: "john@example.com"
      },
      status: "completed",
      date: serverTimestamp()
    }
    ```
  </Step>

  <Step title="Explore Advanced Features">
    Now that you've completed the basics, explore advanced features:

    ### Financial Conciliation
    Reconcile transactions between bank statements and platform records:

    ```javascript
    import { 
      conciliation_data,
      noconciliation_data_b,
      conciliation_summary 
    } from 'src/guards/firebase/Firebase';

    // Get conciliation data
    const conciled = await conciliation_data({ startDate, endDate });
    
    // Get non-conciled bank records
    const notConciled = await noconciliation_data_b({ startDate, endDate });
    
    // Generate summary
    const summary = await conciliation_summary({ startDate, endDate });
    ```

    ### Analytics Dashboard
    Access real-time analytics using Cloud Functions:

    ```javascript
    import { 
      sold_by_office,
      total_sales,
      general_platform_data 
    } from 'src/guards/firebase/Firebase';

    // Sales by office
    const officeSales = await sold_by_office({ eventId });
    
    // Total sales for period
    const sales = await total_sales({ startDate, endDate });
    
    // Platform-wide metrics
    const metrics = await general_platform_data();
    ```

    ### Permission Management
    TMT uses CASL for ability-based access control:

    ```javascript src/App.jsx
    import { AbilityContext } from './guards/contexts/AbilityContext';
    import { defineAbilitiesFor } from './guards/contexts/DefineAbilities';

    const { user } = useAuth();
    const ability = defineAbilitiesFor(user);

    // Check permissions
    if (ability.can('create', 'Event')) {
      // User can create events
    }
    ```
  </Step>
</Steps>

## Common Operations

### Using Redux State Management

TMT uses Redux Toolkit with Redux Persist for state management:

```javascript
import { useSelector, useDispatch } from 'react-redux';
import { fetchEvents } from 'src/store/apps/events/EventsSlice';

function MyComponent() {
  const dispatch = useDispatch();
  const events = useSelector((state) => state.events.events);
  const user = useSelector((state) => state.auth.user);

  useEffect(() => {
    dispatch(fetchEvents());
  }, [dispatch]);

  return <div>{/* Your component */}</div>;
}
```

### Authentication Context

Access authentication methods throughout your app:

```javascript src/guards/firebase/FirebaseContext.js

function MyComponent() {
  const {

  const handleSignOut = async () => {

}
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Authentication errors">
    If you're experiencing authentication issues:
    
    1. Verify Firebase configuration in `Firebase.js`
    2. Ensure the user exists in the `u_staff` collection with `status: true`
    3. Check that the `validate_user_platform` Cloud Function is deployed
    4. Review browser console for specific Firebase error codes
  </Accordion>

  <Accordion title="State not persisting">
    If Redux state isn't persisting across sessions:
    
    1. Check Redux Persist configuration in `Store.js`
    2. Verify the `whitelist` includes necessary reducers
    3. Clear browser local storage and try again
    4. Ensure no errors in the Redux middleware chain
  </Accordion>

  <Accordion title="Cloud Functions not working">
    If Cloud Functions aren't responding:
    
    1. Verify functions are deployed: `firebase deploy --only functions`
    2. Check function names match imports in `Firebase.js`
    3. Review Cloud Functions logs in Firebase Console
    4. Ensure proper IAM permissions are configured
  </Accordion>
</AccordionGroup>

<Note>
  For additional help, check the platform logs in the browser console and Firebase Console.
</Note>
