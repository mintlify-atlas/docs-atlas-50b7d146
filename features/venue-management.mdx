The Venue Management system allows you to create and manage event venues (salones de eventos) with integrated seating chart capabilities powered by Seats.io.

## Overview

Venues are physical locations where events take place. Each venue can have:

- Basic information (name, location, capacity)
- Address and contact details
- Seats.io seating charts for visual layout
- Multiple zones and sections
- Associated events history

<Note>
  Venue management requires specific permissions. The "Creador de Sala de Eventos" role is dedicated to venue creation and management.
</Note>

### Venue Data Structure

Venues are stored in the `event_venues` Firestore collection:

```javascript
{
  name: string,
  description: string,
  venue_type: string,
  // Address information
  address: {
    line: string,
    city: string,
    country: object,
    zip_code: string
  },
  // Capacity
  capacity: number,
  layout_type: string, // 'seated', 'standing', 'mixed'
  // Seats.io integration
  seatsio_chart_key: string,
  seatsio_workspace_key: string,
  // Metadata
  date: {
    created: timestamp,
    updated: timestamp
  },
  status: boolean,
  // Statistics
  events_count: number,
  total_revenue: number
}
```

## Seats.io Integration

The TMT platform integrates with Seats.io for advanced seating chart management.

### Listing Available Charts

```javascript
// From the codebase pattern
const charts = await list_charts_seatsio();

// Returns array of charts:
[
  {
    key: string,
    name: string,
    status: string,
    publishedVersionThumbnailUrl: string
  }
]
```

### Creating Seats.io Events

When an event is created using this venue:

```javascript
const seatsioEvent = await create_event_seatsio({
  chartKey: venue.seatsio_chart_key,
  eventKey: generatedEventKey
});
```

### Seating Chart Visualization

The `SeatsioMapViewer` component displays interactive seating charts:


## Venue Capacity Management

The system tracks capacity at multiple levels:

1. **Total Venue Capacity**: Maximum number of attendees
2. **Zone Capacity**: Capacity per zone/section
3. **Available Capacity**: Real-time tracking during ticket sales
4. **Reserved Capacity**: Tickets held but not purchased

```javascript
// Capacity calculation example
const availableCapacity = venue.capacity - tickets_sold - tickets_reserved;
```

## Venue Status

Venues can be active or inactive:

- **Active (`status: true`)**: Available for event creation
- **Inactive (`status: false`)**: Hidden from venue selection, existing events unaffected

## Best Practices

<Check>
  **Venue Management Best Practices:**
  - Create detailed seating charts in Seats.io before venue creation
  - Test seating chart visualization before assigning to events
  - Keep venue capacity up to date with fire code regulations
  - Use consistent zone naming across venues
  - Document accessibility features in venue description
  - Regular audit of venue information accuracy
  - Archive inactive venues rather than deleting them
  - Maintain historical event data for reporting
</Check>

## Seats.io Best Practices

<Accordion title="Chart Design Guidelines">
  When creating seating charts in Seats.io:
  - Use clear, consistent naming for categories/zones
  - Include row and seat numbers for seated venues
  - Mark accessible seating clearly
  - Test chart responsiveness on mobile devices
  - Use colors that match your branding
  - Include stage/performer area for orientation
  - Add entrance/exit markers for wayfinding
</Accordion>

<Accordion title="Integration Considerations">
  Technical considerations:
  - Each venue should have a unique Seats.io chart
  - Events create copies of charts (don't modify originals)
  - Chart keys are immutable after venue creation
  - Workspace keys are stored securely
  - API rate limits apply to chart operations
</Accordion>

## Troubleshooting

<Accordion title="Seats.io Chart Not Displaying">
  **Problem**: Seating chart fails to load in venue preview
  
  **Solutions**:
  - Verify the chart key is correct in Seats.io dashboard
  - Check workspace key authentication
  - Ensure chart is published in Seats.io
  - Check browser console for API errors
  - Verify network connectivity to Seats.io servers
</Accordion>

<Accordion title="Venue Capacity Mismatch">
  **Problem**: Venue capacity doesn't match seating chart
  
  **Solutions**:
  - Recalculate capacity from Seats.io chart zones
  - Account for standing room if mixed layout
  - Check for disabled seats in chart
  - Verify zone capacities sum to total
  - Update venue capacity after chart modifications
</Accordion>
