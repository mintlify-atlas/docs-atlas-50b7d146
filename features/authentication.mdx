### User Session Management

```javascript
// From src/guards/firebase/FirebaseContext.js:24-74
useEffect(() => {
  const unsubscribe = firebase.auth().onAuthStateChanged(async (user) => {
    if (user) {
      // Fetch user data from Firestore
      const querySnapshot = await Firestore.collection('u_staff').doc(user.uid).get();
      
      // Check user status
      if (querySnapshot.data()?.status !== true) {
        logout();
      } else {
        // Get IP location
        const queryIP = await fetch('https://ipapi.co/json/');
        const data = await queryIP.json();
        
        // Update last access
        await Firestore.collection('u_staff').doc(user.uid).update({
          "date.last_access": firebase.firestore.FieldValue.serverTimestamp(),
          last_IP: data.ip
        });
        
        // Load setup configuration
        dispatch(fetchSetup());
        
        // Set authenticated state
        dispatch(authStateChanged({
          isAuthenticated: true,
          user: {
            uid: user.uid,
            avatar: querySnapshot.data()?.image ?? null,
            account_type: querySnapshot.data()?.account_type,
            phone: querySnapshot.data()?.phone ?? "",
            address: querySnapshot.data()?.address ?? {...},
            email: user.email,
            country_calling_code: data.country_calling_code,
            name: querySnapshot.data()?.name,
            ...(querySnapshot.data())
          },
        }));
      }
    } else {
      dispatch(authStateChanged({
        isAuthenticated: false,
        user: null,
      }));
    }
  });
  
  return () => unsubscribe();
}, [dispatch]);
```

<Note>
  On each login, the system:
  1. Verifies user status is active
  2. Fetches user's IP address and location
  3. Updates last access timestamp
  4. Loads platform setup configuration
  5. Dispatches user data to Redux store
</Note>

### Auth0 Context Setup

```javascript
// From src/guards/auth0/Auth0Context.js:52-86
useEffect(() => {
  const initialize = async () => {
    try {
      auth0Client = new Auth0Client({
        domain: auth0Config.domain,
        clientId: auth0Config.client_id,
        authorizationParams: { redirect_uri: window.location.origin },
      });
      
      await auth0Client.checkSession();
      
      const isAuthenticated = await auth0Client.isAuthenticated();
      
      if (isAuthenticated) {
        const user = await auth0Client.getUser();
        
        dispatch({
          type: 'INITIALIZE',
          payload: { isAuthenticated, user },
        });
      } else {
        dispatch({
          type: 'INITIALIZE',
          payload: { isAuthenticated, user: null },
        });
      }
    } catch (err) {
      console.error(err);
      dispatch({
        type: 'INITIALIZE',
        payload: { isAuthenticated: false, user: null },
      });
    }
  };
  
  initialize();
}, []);
```

### Permission Guard

**Location**: `src/guards/authGuard/PermissionGuard.js`

Protects routes and components based on user permissions:

```javascript
// From src/guards/authGuard/PermissionGuard.js:1-18
import React from "react";
import { useNavigate } from 'react-router-dom';
import { AbilityContext } from "../contexts/AbilityContext";

const PermissionGuard = ({ children, action, subject }) => {
    const ability = React.useContext(AbilityContext);
    const navigate = useNavigate();
    
    React.useEffect(() => {
        if (!ability.can(action, subject)) {
            navigate("/auth/permissions", { replace: true })
        }
    }, [navigate])
    
    return children
};

export default PermissionGuard;
```

**Usage:**

```jsx
<PermissionGuard action="create" subject="events">
  <CreateEventButton />
</PermissionGuard>
```

## Permission System (CASL)

The platform uses [CASL](https://casl.js.org/) for ability-based permission management.

### Ability Context

**Location**: `src/guards/contexts/AbilityContext.js`

Provides ability checking throughout the application.

### Define Abilities

**Location**: `src/guards/contexts/DefineAbilities.js`

Defines permissions for each user role:

```javascript
// From src/guards/contexts/DefineAbilities.js:1-8
import { AbilityBuilder, Ability } from '@casl/ability';

export function defineAbilitiesFor(user) {
    const { can, cannot, build } = new AbilityBuilder(Ability);
    
    if (!user) {
        return build();
    };
```

### Permission Structure

Permissions are defined as action-subject pairs:

- **Action**: What the user wants to do (e.g., 'create', 'edit', 'view', 'delete')
- **Subject**: What the action applies to (e.g., 'events', 'users', 'tickets')

### Session Management

- Sessions are maintained using Firebase Auth tokens
- Tokens are automatically refreshed
- Invalid tokens result in automatic logout
- Session expires after period of inactivity

### Password Security

- Minimum 6 characters required
- Passwords are hashed by Firebase Authentication
- Password reset via email verification
- No plaintext password storage

## Best Practices

## Troubleshooting

<Accordion title="Permission Denied Redirect">
  **Problem**: User is redirected to `/auth/permissions` when accessing a page
  
  **Cause**: User lacks required permission for the action/subject
  
  **Solution**:
  - Verify user's `account_type` is correct
  - Check permissions in `DefineAbilities.js`
  - Ensure PermissionGuard is using correct action/subject
  - Check if user status is active
</Accordion>

<Accordion title="User Automatically Logged Out">
  **Problem**: User is logged out immediately after login
  
  **Cause**: User status is set to `false` (inactive)
  
  **Solution**:
  - Check user document in Firestore
  - Verify `status` field is `true`
  - Update status if user should have access
</Accordion>

<Accordion title="Social Login Not Working">
  **Problem**: Google/Facebook/Twitter login fails
  
  **Solutions**:
  - Verify Firebase project has social auth enabled
  - Check OAuth credentials are configured
  - Verify redirect URIs are whitelisted
  - Check browser console for error messages
</Accordion>
