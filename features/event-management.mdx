The Event Management system is the core of the TMT platform, enabling clients and staff to create, configure, and monitor events from creation through completion.

## Event Lifecycle

<Steps>
  <Step title="Event Creation">
    Create an event with basic information, client assignment, venue selection, and contract linking
  </Step>
  <Step title="Event Configuration">
    Set up zones, seating, pricing, and financial splits
  </Step>
  <Step title="Ticket Generation">
    Generate tickets based on seating configuration
  </Step>
  <Step title="Event Monitoring">
    Track sales, orders, and revenue in real-time
  </Step>
  <Step title="Event Completion">
    Process final transactions and payouts
  </Step>
</Steps>

## Creating Events

### Event Creation UI Flow

**Location**: `src/views/events/new-events/NewEvents.js`

The event creation form collects comprehensive information:

1. **Basic Information**
   - Event name
   - Event type (from setup configuration)
   - Event description
   - Event image (max 2MB)

2. **Client & Contract Assignment**
   - Select client (event organizer)
   - Link existing contract
   - Link legal contract
   - Upload terms of service PDF

3. **Venue Selection**
   - Choose event venue
   - Select Seats.io chart from venue
   - Preview seating layout

4. **Date & Time**
   - Event start date/time
   - Event end date/time
   - Timezone handling

### Event Creation Implementation

```javascript
// From src/components/apps/events/NewEventForm.js:51-75
const searchContracts = async (client_id) => {
  const querySnapshot = await Firestore.collection('contracts')
    .where('user_id', '==', client_id).get();
  const querySnapshot2 = await Firestore.collection('contracts_legal')
    .where('client_id', '==', client_id).get();

  const ContractsData = querySnapshot.docs.map((doc) => {
    const data = doc.data();
    return {
      value: doc.id,
      name: data.name
    };
  });

  const ContractsLegalData = querySnapshot2.docs.map((doc) => {
    const data = doc.data();
    return {
      value: { id: doc.id, doc: data },
      name: data.legal_name
    };
  });

  setContracts(ContractsData);
  setContractsLegal(ContractsLegalData);
};
```

<Note>
  When a client is selected, the system automatically fetches their associated contracts from both the `contracts` and `contracts_legal` collections to enable contract linking.
</Note>

## Event Data Structure

Events are stored in the `events` Firestore collection:

```javascript
{
  name: string,
  event_type: string,
  description: string,
  image: string (URL),
  client_id: string,
  client_name: string,
  contract_id: string,
  contract_legal_id: string,
  venue_id: string,
  venue_name: string,
  seatsio_chart_key: string,
  seatsio_event_key: string,
  event_start_date: timestamp,
  event_end_date: timestamp,
  status: string,
  terms_of_service_url: string,
  date: {
    created: timestamp,
    updated: timestamp
  },
  // Financial data
  revenue: number,
  tickets_sold: number,
  tickets_total: number
}
```

## Best Practices

<Check>
  **Event Management Best Practices:**
  - Always link events to valid contracts
  - Upload event images under 2MB
  - Configure zones and pricing before ticket sales
  - Set up financial splits before accepting payments
  - Test seating chart integration before going live
  - Send test notifications before bulk sends
  - Monitor revenue and sales regularly
  - Update event status as it progresses
</Check>
