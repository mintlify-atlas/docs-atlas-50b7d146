## Staff Management

Staff are internal users who operate the platform. The system supports multiple staff hierarchy levels with different permissions.

### Staff Hierarchy Levels

<Accordion title="Administrador (Administrator)">
  Full system access with the ability to:
  - Manage all users (Staff, Clients, Collaborators)
  - Create and configure events
  - Access all financial data
  - Modify system settings
  - Change event and ticket statuses
  
  See `src/guards/contexts/DefineAbilities.js:288-289` for permission implementation.
</Accordion>

<Accordion title="Coordinador (Coordinator)">
  Event and client management with permissions to:
  - Create and edit Clients and Collaborators
  - Create and edit events
  - Manage event venues
  - Access contracts and addendums
  - View tickets and credentials
  - Cannot: create Staff, change event/ticket status, access payouts
  
  See `src/guards/contexts/DefineAbilities.js:11-92` for full permission list.
</Accordion>

<Accordion title="Contador (Accountant)">
  Financial and reporting access:
  - View clients and contracts
  - Access events and transaction data
  - View financial reports
  - Cannot: create users, events, or modify configurations
  
  See `src/guards/contexts/DefineAbilities.js:93-161` for full permission list.
</Accordion>

<Accordion title="Soporte (Support)">
  Read-only support access:
  - View events and details
  - Access ticket information
  - View client and collaborator details
  - Cannot: create or edit most entities
  
  See `src/guards/contexts/DefineAbilities.js:162-230` for full permission list.
</Accordion>

<Accordion title="Creador de Sala de Eventos (Venue Creator)">
  Specialized role for venue management:
  - Create and edit event venues
  - Manage venue seating charts
  - View venue details and events
  - Cannot: access other modules
  
  See `src/guards/contexts/DefineAbilities.js:231-285` for full permission list.
</Accordion>

### Creating Staff Users

Staff users are created through the staff management interface.

**UI Flow:**
1. Navigate to Staff List (`src/views/Users/Staff/StaffTable.js`)
2. Click "Create Staff" button
3. Fill out the New Staff form (`src/views/Users/NewUsers/NewUsersStaff.js`)
4. Upload profile image (optional)
5. Submit to create user account

**Implementation:**

```javascript
// From src/components/apps/users/NewUsers/NewUserStaffForm.js:103-142
const userData = {
  name: values.name,
  commercial_name: "",
  email: values.email,
  phone: values.phone,
  password: values.password,
  status: true,
  id: values.id,
  id_type: values.id_type
};

let additionalData = {
  city: values.city,
  account_type: values.account_type,
  country: locations.find(c => c.code === values.country),
  line: values.line,
  zip_code: values.zip_code,
};

const register = await create_staff({ ...userData, ...additionalData });

if (imageReal) {
  const storageRef = ref(Storage, `u_staff/${register.data.uid}/profile_picture.jpg`);
  await uploadBytes(storageRef, imageReal);
  imageUrl = await getDownloadURL(storageRef);
}

await Firestore.collection(`u_staff`).doc(register.data.uid).update({ image: imageUrl });
```

**Required Fields:**
- Email (validated)
- Name
- Phone number (validated with MUI Tel Input)
- ID number and type
- Address (line, city, zip code, country)
- Account type (hierarchy level)
- Password (min 6 characters, with confirmation)

**Data Storage:**
Staff data is stored in the `u_staff` Firestore collection with the following structure:

```javascript
{
  name: string,
  email: string,
  phone: string,
  status: boolean,
  id: string,
  id_type: string,
  city: string,
  account_type: string,
  country: object,
  line: string,
  zip_code: string,
  image: string (URL),
  date: {
    created: timestamp,
    last_access: timestamp
  },
  last_IP: string
}
```

2. Click "Create Client" button
3. Fill out the New Client form (`src/views/Users/NewUsers/NewUsersClient.js`)
4. Add legal information and bank accounts
5. Submit to create client account

```javascript
// Similar to staff creation but uses create_client function
```

**Client Data Structure:**
Clients have additional fields for legal and financial information:

```javascript
{
  name: string,
  commercial_name: string,
  email: string,
  phone: string,
  status: boolean,
  id: string,
  id_type: string,
  // Address information
  city: string,
  country: object,
  line: string,
  zip_code: string,
  // Legal information
  legal_name: string,
  legal_id: string,
  // Bank accounts array
  bank_accounts: [{
    account_number: string,
    bank_name: string,
    account_type: string
  }],
  image: string (URL)
}
```

### Client Details View

The client detail view (`src/views/Users/Clients/clients-detail/ClientsDetail.js`) displays:

- **Profile Tab**: Basic information, contact details, legal info
- **Events Tab**: List of client's events with revenue data
- **Contracts**: Associated contracts and legal documents
- **Bank Accounts**: Payment information for payouts
- **Payouts**: Client payout accounts and history

## User Status Management

All user types support status management:

- **Active (`status: true`)**: User can log in and access the system
- **Inactive (`status: false`)**: User is blocked from logging in

<Warning>
  When a staff user's status is set to `false`, they are automatically logged out. See `src/guards/firebase/FirebaseContext.js:28-31` for implementation.
</Warning>

```javascript
// From src/guards/firebase/FirebaseContext.js:28-31
const querySnapshot = await Firestore.collection('u_staff').doc(user.uid).get();

if (querySnapshot.data()?.status !== true) {
  logout();
}
```

## Access Tracking

The system tracks user access for security purposes:

```javascript
// From src/guards/firebase/FirebaseContext.js:33-40
const queryIP = await fetch('https://ipapi.co/json/');
const data = await queryIP.json();

await Firestore.collection('u_staff').doc(user.uid).update({
  "date.last_access": firebase.firestore.FieldValue.serverTimestamp(),
  last_IP: data.ip
});
```

Each staff login records:
- Last access timestamp
- IP address
- Country calling code from IP location

## Profile Image Management

All user types support profile image uploads:

<Note>
  Images are limited to 2MB and stored in Firebase Storage under the pattern:
  - Staff: `u_staff/{uid}/profile_picture.jpg`
  - Clients: `u_clients/{uid}/profile_picture.jpg`
  - Collaborators: `u_collaborators/{uid}/profile_picture.jpg`
</Note>

## User Editing

- **Edit Staff**: `src/views/Users/EditUsers/EditUsersStaff.js`

## Best Practices

<Check>
  **User Creation Best Practices:**
  - Always validate email addresses before creating users
  - Enforce strong passwords (minimum 6 characters)
  - Verify phone numbers using the MUI Tel Input validator
  - Upload profile images under 2MB
  - Assign appropriate account types based on responsibilities
  - Regularly review and update user statuses
</Check>
