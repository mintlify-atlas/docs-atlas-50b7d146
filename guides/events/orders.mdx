# Managing Event Orders

Track and manage all ticket orders for your events, from initial purchase through fulfillment and entry. Monitor sales performance, process refunds, and handle customer inquiries.

## Overview

The order management system allows you to:

- **View all orders** for an event
- **Track order status** and payment
- **Review order details** including tickets and transactions
- **Monitor sales performance** in real-time
- **Export order data** for reporting and accounting
- **Process refunds** and modifications

Navigate to **Events** → Select your event → **Órdenes**

## Order Actions

### Viewing Order Receipt

Generate and display order receipt:

```javascript
const generateReceipt = async (orderId) => {
  const order = await Firestore.collection('orders').doc(orderId).get();
  const orderData = order.data();
  
  // Generate PDF or HTML receipt
  const receipt = {
    order_id: orderId,
    customer: orderData.customer,
    items: orderData.tickets.map(t => ({
      description: `${t.zone_name} - ${t.seat}`,
      quantity: 1,
      unit_price: t.price,
      total: t.price
    })),
    subtotal: orderData.amount,
    fees: calculateFees(orderData.amount),
    total: orderData.amount,
    payment_method: orderData.transactions[0].payment_name
  };
  
  return receipt;
};
```

### Processing Refunds

Handle order cancellations and refunds:

```javascript
const processRefund = async (orderId, reason) => {
  // Update order status
  await Firestore.collection('orders').doc(orderId).update({
    status: false,
    'ledger': firebase.firestore.FieldValue.arrayUnion({
      action: 'Reembolsado',
      reason: reason,
      date: new Date()
    })
  });
  
  // Cancel associated tickets
  const order = await Firestore.collection('orders').doc(orderId).get();
  const tickets = order.data().tickets;
  
  for (const ticket of tickets) {
    await Firestore
      .collection('events')
      .doc(order.data().event_id)
      .collection('tickets')
      .doc(ticket.id)
      .update({
        status: false,
        cancelled_date: firebase.firestore.FieldValue.serverTimestamp()
      });
  }
  
  // Process payment refund through gateway
  const transaction = order.data().transactions[0];
  await processPaymentRefund(transaction.payment_id, transaction.amount);
};
```

<Warning>
Refunds cannot be undone. Always verify the request before processing a refund.
</Warning>

## Order Analytics

Track order metrics:

```javascript
const getOrderAnalytics = async (eventId) => {
  const orders = await Firestore
    .collection('orders')
    .where('event_id', '==', eventId)
    .where('status', '==', true)
    .get();
  
  const analytics = {
    total_orders: orders.size,
    total_revenue: 0,
    total_tickets: 0,
    avg_order_value: 0,
    orders_by_zone: {},
    orders_by_date: {},
    payment_methods: {}
  };
  
  orders.forEach(doc => {
    const order = doc.data();
    analytics.total_revenue += order.amount;
    analytics.total_tickets += order.tickets.length;
    
    // Group by zone
    order.tickets.forEach(ticket => {
      if (!analytics.orders_by_zone[ticket.zone_name]) {
        analytics.orders_by_zone[ticket.zone_name] = 0;
      }
      analytics.orders_by_zone[ticket.zone_name]++;
    });
    
    // Track payment methods
    const payment = order.transactions[0].payment_name;
    if (!analytics.payment_methods[payment]) {
      analytics.payment_methods[payment] = 0;
    }
    analytics.payment_methods[payment]++;
  });
  
  analytics.avg_order_value = 
    analytics.total_revenue / analytics.total_orders;
  
  return analytics;
};
```

## Troubleshooting

### Order Not Found

If an order doesn't appear:

1. **Verify order ID** - Check for typos in search
2. **Check filters** - Clear all filters and search again
3. **Confirm event** - Ensure you're viewing the correct event
4. **Review status** - Inactive orders may be hidden by default

### Payment Issues

If a payment shows as pending:

<Warning>
1. **Check payment gateway** - Review transaction status in Stripe/PayPal
2. **Verify webhook delivery** - Ensure payment confirmations were received
3. **Manual verification** - Contact payment provider for transaction status
4. **Update manually** - If confirmed, update order status in database
</Warning>

### Missing Tickets

If tickets don't appear in order:

1. Check that ticket generation completed successfully
2. Verify tickets exist in event tickets collection
3. Confirm ticket IDs match order ticket array
4. Review order creation logs for errors

## Best Practices

### Order Management

- **Review pending orders** daily to catch payment issues
- **Export order data** regularly for backup and accounting
- **Monitor refund requests** and process promptly
- **Track order patterns** to optimize pricing and inventory

### Customer Service

- **Respond quickly** to order inquiries
- **Verify identity** before discussing order details
- **Document interactions** in order ledger
- **Escalate issues** to management when necessary

### Data Security

- **Protect customer data** - Limit access to authorized personnel only
- **Secure payment information** - Never store full card numbers
- **Audit access logs** - Track who views sensitive order data
- **Comply with regulations** - Follow PCI DSS and data privacy laws
