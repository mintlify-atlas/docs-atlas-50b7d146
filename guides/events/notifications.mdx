# Event Notifications

Communicate with event attendees, staff, and stakeholders through the TMT notification system. Send real-time updates, alerts, and important information directly to users' devices.

## Overview

The notification system enables you to:

- **Send push notifications** to attendees and staff
- **Target specific audiences** based on ticket type or credentials
- **Schedule notifications** for future delivery
- **Track notification delivery** and engagement
- **Send emergency alerts** for critical situations

## Accessing Notifications

Navigate to **Events** → Select your event → **Notificaciones**

## Notification Types

### General Announcements

Broadcast messages to all event participants:
- Event schedule changes
- General information updates
- Welcome messages
- Post-event thank you messages

### Targeted Notifications

Send messages to specific groups:
- Zone-specific updates (e.g., "VIP lounge now open")
- Credential-based alerts (e.g., staff call times)
- Time-sensitive reminders

### Emergency Alerts

Critical notifications for urgent situations:
- Safety alerts
- Evacuation notices
- Weather warnings
- Event cancellations

## Notification Delivery System

Notifications are sent through Firebase Cloud Messaging:

```javascript
// Notification creation and dispatch
const sendEventNotification = async (notificationData) => {
  const { title, body, audience, eventId, priority } = notificationData;
  
  // Get recipient tokens based on audience
  const recipientTokens = await getRecipientTokens(audience, eventId);
  
  // Create notification payload
  const payload = {
    notification: {
      title: title,
      body: body,
      priority: priority || 'normal',
      sound: priority === 'urgent' ? 'alert' : 'default',
      badge: '1'
    },
    data: {
      event_id: eventId,
      notification_type: 'event_update',
      timestamp: new Date().toISOString()
    }
  };
  
  // Send to FCM
  const response = await admin.messaging().sendToDevice(
    recipientTokens,
    payload
  );
  
  // Log notification
  await Firestore.collection('events')
    .doc(eventId)
    .collection('notifications')
    .add({
      title,
      body,
      count: recipientTokens.length,
      date_created: firebase.firestore.FieldValue.serverTimestamp(),
      status: 'sent',
      delivery_stats: response
    });
  
  return response;
};
```

## Recipient Token Management

Fetch device tokens for targeted delivery:

```javascript
const getRecipientTokens = async (audience, eventId) => {
  let tokens = [];
  
  switch (audience) {
    case 'all':
      // Get all users with event access
      const allUsers = await Firestore
        .collection('user_event_access')
        .where('event_id', '==', eventId)
        .get();
      tokens = allUsers.docs.map(doc => doc.data().fcm_token);
      break;
      
    case 'ticket_holders':
      // Get users who purchased tickets
      const orders = await Firestore
        .collection('orders')
        .where('event_id', '==', eventId)
        .where('status', '==', true)
        .get();
      const userIds = orders.docs.map(doc => doc.data().user_id);
      const users = await Firestore
        .collection('users')
        .where(firebase.firestore.FieldPath.documentId(), 'in', userIds)
        .get();
      tokens = users.docs.map(doc => doc.data().fcm_token);
      break;
      
    case 'credentials':
      // Get credential holders
      const credentials = await Firestore
        .collection('events')
        .doc(eventId)
        .collection('credentials')
        .where('status', '==', true)
        .get();
      tokens = credentials.docs.map(doc => doc.data().holder?.fcm_token);
      break;
      
    default:
      tokens = [];
  }
  
  return tokens.filter(token => token); // Remove null/undefined
};
```

## Notification Delivery Tracking

Monitor notification performance:

```javascript
const getNotificationStats = async (notificationId, eventId) => {
  const notification = await Firestore
    .collection('events')
    .doc(eventId)
    .collection('notifications')
    .doc(notificationId)
    .get();
  
  const stats = notification.data().delivery_stats;
  
  return {
    sent: stats.successCount || 0,
    failed: stats.failureCount || 0,
    opened: stats.openedCount || 0, // Requires app tracking
    clicked: stats.clickedCount || 0
  };
};
```

## Best Practices

### Timing

- **Send reminders** 24 hours, 2 hours, and 30 minutes before event
- **Avoid late night notifications** (10pm - 8am) unless urgent
- **Schedule non-critical messages** during business hours
- **Space notifications** at least 1 hour apart

### Content

- **Be concise** - Users see limited characters on lock screen
- **Front-load key information** in title and first line
- **Use emoji sparingly** for emphasis or urgency indicators
- **Include clear call-to-action** when applicable
- **Proofread carefully** - notifications cannot be recalled

### Frequency

- **Limit to 3-5 notifications** per event for non-staff
- **More frequent for staff** with operational updates
- **Consider notification fatigue** - quality over quantity
- **Allow users to opt-out** of non-critical notifications

### Targeting

- **Use zone targeting** for large venues with different schedules
- **Segment by ticket type** for VIP experiences
- **Separate staff communications** from attendee messages
- **Test with small group** before mass sending

## Troubleshooting

### Notifications Not Received

If users report not receiving notifications:

<Warning>
1. **Verify app permissions** - Push notifications must be enabled
2. **Check device tokens** - Ensure FCM tokens are valid
3. **Review targeting** - Confirm user is in selected audience
4. **Test notification settings** - Some devices have aggressive battery savers
</Warning>

### Low Open Rates

If notifications have poor engagement:

1. **Review content** - Is the title compelling?
2. **Check timing** - Are you sending at optimal times?
3. **Reduce frequency** - Are you over-notifying?
4. **Improve targeting** - Are messages relevant to recipients?

### Delivery Failures

If notifications fail to send:

1. Check Firebase Cloud Messaging quota
2. Verify API keys are valid
3. Review error logs in Firebase console
4. Ensure recipient tokens are current
