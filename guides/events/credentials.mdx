# Event Credentials

Control access to your events by creating and managing digital credentials for staff, vendors, artists, VIP guests, and other authorized personnel.

## Overview

The credentials system allows you to:

- **Issue digital access passes** for event personnel
- **Define access levels** and permissions
- **Track credential status** and usage
- **Manage holder information** and contact details
- **Export credential lists** for security checkpoints

## Accessing Credential Management

Navigate to **Events** → Select your event → **Credenciales**

## Credential Types

TMT supports various credential access levels:

- **Staff** - Event staff and operations team
- **Security** - Security personnel and guards
- **Vendor** - Food, beverage, and merchandise vendors
- **Artist** - Performers and entertainers
- **VIP** - VIP guests and sponsors
- **Press** - Media and journalists
- **Producer** - Event producers and organizers
- **Technical** - Technical crew and support

## Managing Credentials

### Activating/Deactivating Credentials

Change credential status to control access:

```javascript
await Firestore.collection('events')
  .doc(eventId)
  .collection('credentials')
  .doc(credentialId)
  .update({
    status: false, // Deactivate
    'ledger': firebase.firestore.FieldValue.arrayUnion({
      action: 'Desactivado',
      date: new Date(),
      by: currentUser.uid
    })
  });
```

<Warning>
Deactivating a credential immediately revokes access. The holder will not be able to enter restricted areas until the credential is reactivated.
</Warning>

### Editing Credential Information

Update holder information or access level:

1. Click the credential to open details
2. Click **Editar**

## Access Control Integration

Credentials can be validated at entry points:

### QR Code Scanning

Each credential has a unique QR code that can be scanned:

```javascript
// Credential validation endpoint
const validateCredential = async (credentialId, eventId) => {
  const credential = await Firestore
    .collection('events')
    .doc(eventId)
    .collection('credentials')
    .doc(credentialId)
    .get();
  
  if (!credential.exists) {
    return { valid: false, reason: 'Credential not found' };
  }
  
  const data = credential.data();
  
  if (!data.status) {
    return { valid: false, reason: 'Credential inactive' };
  }
  
  // Log check-in
  await logCheckIn(credentialId, eventId);
  
  return { valid: true, holder: data.holder, access: data.access };
};
```

### Check-In Tracking

Track when and where credentials are used:

```javascript
const logCheckIn = async (credentialId, eventId) => {
  await Firestore
    .collection('events')
    .doc(eventId)
    .collection('credentials')
    .doc(credentialId)
    .update({
      'check_ins': firebase.firestore.FieldValue.arrayUnion({
        timestamp: new Date(),
        location: 'Main Entrance',
        scanner_id: 'SCAN-001'
      })
    });
};
```

## Security Best Practices

### Credential Issuance

- **Verify identity** before issuing credentials
- **Use unique identifiers** for each holder
- **Set expiration dates** for temporary access
- **Review access levels** regularly

### Access Control

- **Limit credential creation** to authorized administrators
- **Audit credential usage** through check-in logs
- **Revoke immediately** when access should end
- **Use status field** instead of deleting credentials

### Data Protection

- **Collect only necessary** personal information
- **Secure credential data** with proper Firebase rules
- **Limit PII exposure** in exports and reports
- **Comply with privacy** regulations (GDPR, local laws)

## Troubleshooting

### Credential Not Scanning

If a QR code won't scan:

1. Check credential status is "Activo"
2. Verify QR code is not damaged or obscured
3. Ensure scanner app has latest credentials
4. Try manual ID entry as backup

### Cannot Create Credential

If credential creation fails:

<Warning>
- Verify all required fields are completed
- Check that email format is valid
- Ensure event is in correct status
- Confirm user has proper permissions
</Warning>

### Duplicate Credentials

To prevent duplicate credentials:

1. Search by holder ID before creating
2. Use unique identification numbers
3. Review existing credentials first
4. Deactivate old credentials instead of creating new ones
