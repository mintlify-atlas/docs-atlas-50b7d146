# Processing Payouts

Payout accounts in TMT define the bank accounts and payment methods used to disburse funds to event organizers, vendors, and other beneficiaries.

## Overview

Payout methods are stored in the `payout_methods/tmt/methods` Firestore collection path and configure:

- Beneficiary account details
- Payment method type (digital, cash, check)
- Currency and country information
- Bank account metadata
- Status and creation timestamps

## Payout Account Structure

```javascript
{
  name: string,              // Payout account name
  type: string,              // Payment type (digital, cash, check)
  currency: Object,          // Currency object with id and name
  country: string,           // Country code
  method: string,            // Payment method name
  data: Object,              // Method-specific metadata
  status: boolean,           // Active/inactive status
  date: {
    created: Timestamp,      // Creation timestamp
    updated: Timestamp       // Last update timestamp
  }
}
```

## Payment Method Types

### Digital Payments

Digital payment methods require specific data fields:

- **Bank Transfers**: Bank name, account number, account type
- **Mobile Wallets**: Phone number, wallet provider
- **Online Payment Services**: Email, account ID

The system dynamically loads available banks based on country:

```javascript
// From NewPayoutForm.js:156-168
if (field.value === 'bank') {
  return {
    ...item,
    data: banks[selectedValues.country]
  };
}
```

### Cash Payments

Cash payout methods typically require:
- Recipient name
- ID document type and number
- Collection location

### Check Payments

Check payout methods need:
- Payee name
- Mailing address
- ID verification

<Warning>

</Warning>

## Managing Account Status

Payout accounts can be activated or deactivated:

- **Active** (`status: true`): Available for disbursements
- **Inactive** (`status: false`): Hidden from payout options

<Warning>
  Deactivating a payout account does not delete historical transaction records. Reactivate the account if needed for future payments.
</Warning>

## Integration with Payment Methods

Payout accounts complement payment methods:

- **Payment Methods**: How customers pay (inbound)
- **Payout Accounts**: Where funds are disbursed (outbound)

Both use similar data structures but serve different purposes in the financial flow.

## Best Practices

1. **Descriptive Names**: Use clear, identifiable names for payout accounts
2. **Verify Bank Details**: Double-check account numbers and routing information
3. **Keep Active**: Only keep currently used payout accounts active
4. **Regular Audits**: Periodically review payout accounts for accuracy
5. **Document Updates**: Note reasons for changes in external records
