# Custody Account Operations

Custody accounts in TMT serve as intermediate holding accounts where customer payments are collected before disbursement. They are essential for financial reconciliation and tracking fund flows.

## Overview

Custody accounts are stored in the `custody_accounts` Firestore collection and represent:

- Bank accounts or digital wallets
- Payment collection points
- Balance tracking mechanisms
- Reconciliation checkpoints

## Custody Account Structure

```javascript
{
  name: string,                      // Account name
  type: string,                      // Account type (digital, cash, check)
  currency: string,                  // Currency code (USD, VEF)
  country: string,                   // Country code
  balance_available: number,         // Available balance
  balance_pending: number,           // Pending balance
  opening_balance: number,           // Initial balance
  opening_balance_available: boolean, // Can use opening balance
  client: {
    client_id: string,               // Client ID
    name: string                     // Client name
  },
  manager: {
    name: string,                    // Account manager name
    email: string,                   // Manager email
    phone: string                    // Manager phone
  },
  metadata: {
    bank: Object,                    // Bank information
    account_number: string,          // Account number
    account_type: Object             // Account type
  },
  status: boolean,                   // Active/inactive
  date: {
    created: Timestamp,              // Creation timestamp
    updated: Timestamp               // Last update timestamp
  }
}
```

## Balance Management

Custody accounts track three types of balances:

### Available Balance

Funds available for withdrawal or disbursement:

```javascript
balance_available: number  // Current available balance
```

Updated when:
- Transactions are confirmed
- Payouts are processed
- Reconciliations are completed

### Pending Balance

Funds awaiting confirmation:

```javascript
balance_pending: number  // Pending transactions
```

Used for:
- Unconfirmed bank transfers
- Processing transactions
- Pending reconciliations

### Opening Balance

Initial account balance:

```javascript
opening_balance: number  // Initial balance
opening_balance_available: boolean  // Can use opening balance
```

<Warning>
  The opening balance should only be set once during account creation. Changes to the opening balance affect all historical reconciliation calculations.
</Warning>

## Account Types and Country Rules

### Venezuelan Accounts (ve)

Require complete bank information:
- Bank name from approved list
- 16+ digit account number
- Account type specification
- Opening balance

### International Accounts

More flexible requirements:
- Bank details optional
- Account number format varies
- Opening balance optional

## Integration with Payment Methods

Custody accounts are linked to payment methods:

```javascript
// Payment methods reference custody accounts
payment_method = {
  name: "Zelle - TMT USD",
  custody_account: {
    id: "custody_account_id",
    name: "TMT USD Collection - Bank of America"
  },
  // ... other fields
}
```

When a transaction uses a payment method:
1. Funds are recorded to the linked custody account
2. Balance is updated (available or pending)
3. Transaction is tracked for reconciliation

## Manager Responsibilities

The account manager is responsible for:

- Monitoring account balances
- Reconciling bank statements
- Processing withdrawals
- Reporting discrepancies
- Maintaining account access

## Account Status Management

Control account availability:

- **Active** (`status: true`): Accepts new transactions
- **Inactive** (`status: false`): No new transactions

Deactivate accounts for:
- Account closure
- Temporary suspension
- Security issues
- Compliance holds

<Warning>
  Deactivating a custody account prevents new transactions but does not affect existing balances or historical records.
</Warning>

## Reconciliation Integration

Custody accounts are central to the reconciliation process:

```javascript
// From SummaryConciliations.js:28-30
const querySnapshot = await Firestore.collection('custody_accounts')
  .where("country", "==", "ve")
  .where("currency", "==", "VEF")
  .where("type", "==", "digital")
  .get();
```

Reconciliation compares:
- Custody account balance (system)
- Bank statement balance (external)
- Transaction records (both sources)

## Best Practices

1. **Accurate Setup**: Verify all bank details during creation
2. **Single Purpose**: One custody account per bank account/payment method combination
3. **Regular Monitoring**: Check balances daily
4. **Prompt Reconciliation**: Reconcile at least monthly
5. **Manager Access**: Ensure manager has actual account access
6. **Balance Verification**: Confirm opening balance matches bank records
7. **Documentation**: Keep external records of account credentials

## Account Naming Convention

Use consistent naming for easy identification:

```
[Client] [Currency] [Type] - [Bank/Provider]
```

Examples:
- "TMT USD Digital - Bank of America"
- "EventCo VEF Digital - Banco Mercantil"
- "TMT Cash Collection - Main Office"
