# Managing Transactions

Transactions in TMT represent the financial operations associated with event ticket purchases. Each transaction is automatically created when an order is processed and captures critical payment information.

## Overview

Transactions are stored in the `orders_transactions` Firestore collection and contain comprehensive details about:

- Payment method used
- Amount and currency
- Associated event and order
- Client information
- Box office (taquilla) details
- Exchange rates for multi-currency transactions

## Transaction Data Structure

Each transaction contains the following fields:

```javascript
{
  id: string,                    // Transaction ID
  order_id: string,              // Associated order ID
  event_name: string,            // Name of the event
  payment_name: string,          // Payment method name
  office_name: string,           // Box office name
  client_name: string,           // Customer name
  amount: number,                // Transaction amount in USD
  amount_currency: string,       // Currency code (USD, VEF)
  amount_exchange: number,       // Amount in local currency
  amount_exchange_rate: number,  // Exchange rate applied
  date: {
    created: Timestamp,          // Creation timestamp
    updated: Timestamp           // Last update timestamp
  }
}
```

## Multi-Currency Support

TMT handles transactions in multiple currencies with automatic conversion:

### VEF (Venezuelan Bolívar) Transactions

When `amount_currency` is set to "VEF":

- `amount_exchange`: Amount in Bolívares
- `amount`: Equivalent amount in USD
- `amount_exchange_rate`: Current exchange rate

```javascript
// Display logic from TransactionDetail.js:186-202
if (transaction.amount_currency === 'VEF') {
  primaryAmount = `Bs. ${transaction.amount_exchange.toFixed(2)}`
  equivalentUSD = `$${transaction.amount.toFixed(2)}`
  exchangeRate = `Bs. ${transaction.amount_exchange_rate.toFixed(2)}`
}
```

### USD Transactions

For USD transactions:
- `amount`: Direct USD amount
- No exchange rate or conversion needed

## API Integration

Transactions are retrieved through the Redux store:

```javascript
// From transactionsSlice.js:35-66
export const fetchTransactions = () => async (dispatch) => {
  const querySnapshot = await Firestore
    .collection('orders_transactions')
    .get();
  
  const TransactionsArray = querySnapshot.docs.map((doc) => ({
    id: doc.id,
    ...doc.data()
  }));
  
  dispatch(getTransactions(TransactionsArray));
};
```

## Best Practices

1. **Regular Reconciliation**: Cross-reference transactions with custody account movements
2. **Currency Validation**: Always verify exchange rates for VEF transactions
3. **Order Verification**: Check the associated order for complete context
4. **Export for Analysis**: Use CSV exports for financial reporting and analysis
