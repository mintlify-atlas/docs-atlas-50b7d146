## Understanding Customers

Customers in TMT are distinct from other user types:

### Customer vs Other User Types

<Tabs>
  <Tab title="Customers">
    **Customers (Buyers)**
    - End users who make purchases
    - May not have platform login access
    - Records created through transaction processing
    - Limited to viewing their own orders (if portal access granted)
    - Managed primarily by staff and collaborators
  </Tab>

  <Tab title="Clients">
    **Clients**
    - Business entities using the platform
    - Full platform access
    - Submit and manage transactions
    - Upload legal documentation
    - Multiple users per organization
  </Tab>

  <Tab title="Collaborators">
    **Collaborators**
    - External users working with clients
    - Platform access based on assignment
    - Process client transactions
    - Limited to assigned client data
  </Tab>

  <Tab title="Staff">
    **Staff**
    - Internal TMT team members
    - Administrative platform access
    - Manage all user types
    - System configuration privileges
  </Tab>
</Tabs>

## Viewing Customer Records

Access and browse customer information through the customer list:

### Accessing the Customer List

<Steps>
  <Step title="Navigate to Customers">
    Go to **Users** > **Customers** in the main navigation menu.
  </Step>

  <Step title="Browse Customer List">
    The customer table displays:
    - Profile picture or avatar
    - Customer name
    - Creation date
    - Account status (Active/Inactive)
    - Email address
    - Action buttons
  </Step>

  <Step title="Use Search and Filters">
    Locate specific customers using:
    - **Search Bar**: Filter by name, email, or creation date
    - **Filter Icon**: Apply advanced filters
    - **Sort Options**: Click column headers to sort
  </Step>
</Steps>

## Exporting Customer Data

Export customer lists for reporting and analysis:

<Steps>
  <Step title="Navigate to Customer List">
    Go to the main customer list page.
  </Step>

  <Step title="Apply Filters (Optional)">
    Use search and filters to narrow down the customer list to your desired criteria.
  </Step>

  <Step title="Export to CSV">
    Click the **Download** icon in the top-right corner of the customer list.
  </Step>

  <Step title="Open Exported File">
    The CSV file downloads containing:
    - Customer names
    - Email addresses
    - Creation dates
    - Status information
    - All visible table data
  </Step>
</Steps>

<Tip>
  Apply filters before exporting to create targeted customer lists for specific reporting needs.
</Tip>

## Customer Status Management

### Understanding Customer Status

Customer status affects their ability to interact with the platform:

<CardGroup cols={2}>
  <Card title="Active Customers" icon="circle-check">
    - Can place new orders
    - Receive email notifications
    - Access customer portal (if enabled)
    - Eligible for promotions
    - Standard service level
  </Card>
  <Card title="Inactive Customers" icon="circle-xmark">
    - Cannot place new orders
    - No email notifications
    - Portal access disabled
    - Excluded from promotions
    - May require reactivation
  </Card>
</CardGroup>

### Common Reasons for Inactive Status

- Customer request for account closure
- Suspicious activity or fraud prevention
- Extended period of inactivity
- Payment issues or disputes
- Administrative review in progress

<Warning>
  Changing customer status may affect their ability to access services and receive communications. Always verify the reason before status changes.
</Warning>

## Pagination and Navigation

Efficiently navigate large customer lists:

### Pagination Controls

- **First Page**: Jump to the beginning of the list
- **Previous Page**: Go back one page
- **Current Page Report**: Shows "From X to Y of Z total"
- **Next Page**: Advance one page
- **Last Page**: Jump to the end of the list
- **Rows Per Page**: Select 10, 15, or 20 customers per page

### Navigation Best Practices

1. **Use Filters First**: Narrow down results before browsing
2. **Adjust Rows Per Page**: Show more items to reduce page navigation
3. **Sort Strategically**: Order by relevant columns (name, date, etc.)
4. **Bookmark Searches**: Save frequently used filter combinations

## Searching and Filtering Tips

<Tabs>
  <Tab title="By Name">
    **Search by Customer Name**
    - Enter full or partial name
    - Search is case-insensitive
    - Matches anywhere in the name field
    - Use quotes for exact matches
  </Tab>

  <Tab title="By Date">
    **Search by Creation Date**
    - Enter date in format: DD/MM/YYYY
    - Find customers registered on specific dates
    - Sort by date column for chronological view
    - Use date range filters if available
  </Tab>

  <Tab title="By Status">
    **Filter by Status**
    - Type "Activo" for active customers
    - Type "Inactivo" for inactive customers
    - Combine with other filters
    - Export filtered results
  </Tab>

  <Tab title="By Email">
    **Search by Email**
    - Enter full or partial email address
    - Useful for duplicate detection
    - Case-insensitive search
    - Matches domain or username
  </Tab>
</Tabs>

## Troubleshooting

<Tabs>
  <Tab title="Cannot Find Customer">
    **Issue**: Customer not appearing in search results
    
    **Solutions**:
    - Clear all filters and try again
    - Check spelling of search terms
    - Try searching by email instead of name
    - Verify customer account exists
    - Check if customer is in different client's records
  </Tab>

  <Tab title="Export Not Working">
    **Issue**: CSV export fails or is empty
    
    **Solutions**:
    - Ensure customers exist in the list
    - Check browser pop-up blocker settings
    - Verify sufficient browser permissions
    - Try exporting smaller date ranges
    - Clear browser cache and retry
  </Tab>

  <Tab title="Slow Performance">
    **Issue**: Customer list loads slowly
    
    **Solutions**:
    - Reduce rows per page setting
    - Apply filters to limit results
    - Clear browser cache
    - Check internet connection
    - Try during off-peak hours
  </Tab>
</Tabs>
